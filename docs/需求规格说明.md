# EduFlow 教学服务管理系统 — 软件需求规格说明（SRS）

**文档性质**：依据当前代码库实现反向归纳的功能与约束说明（规格描述现状及隐含业务意图）。  
**生成日期**：2026-05-03  
**适用代码**：与本仓库 `types/index.ts`、`components/layout/DashboardLayout.tsx`、`lib/storage.ts` 等保持一致。

---

## 1. 引言

### 1.1 编写目的

描述 EduFlow 在教学招生、订单履约、教练接单排课、学管协作、运营审核与财务台账、系统配置等方面的功能边界与角色分工，作为产品沟通、验收与后续接入真实后端的基准。

### 1.2 系统定位

- **名称**：EduFlow（教学服务管理系统）。
- **形态**：Web 应用（Next.js App Router），登录后按角色展示侧边导航与仪表盘。
- **数据**：全部业务数据存储于浏览器 **localStorage**（键前缀 `eduflow:`），首次访问通过 Mock 种子数据初始化；清除浏览器数据将导致业务数据丢失。

### 1.3 范围说明

**包含**：认证与会话、多角色工作台、学员与订单全链路（试课/正课）、接单与市场、日历与课时、课后反馈与家长评价、学习规划书审核、退费双审流程、分公司与用户审核、财务记录、教练收入与汇总、科目与价格与全局开关、订单/教师补录、操作日志、个人设置等。

**不包含（当前仓库层面）**：真实支付网关、服务端权限与审计、跨设备同步、文件对象存储（凭证等多以 Base64 存本地）。

## 2. 用户与权限模型

### 2.1 角色枚举

| 角色代码 | 中文称谓 | 主要职责概要 |
|---------|---------|----------------|
| SALES | 招生老师 | 学员档案、创建试课/正课订单、跟进订单状态与支付凭证 |
| TUTOR | 伴学教练 | 接单中心、我的学员、日历、课后反馈、收入查看、接单设置 |
| MANAGER | 学管 | 与教练类似的接单与学员视图；额外：伴学教练管理、学习规划书管理 |
| OPERATOR | 运营人员 | 订单集中管理、退款审核、用户与分公司、财务记录、教练收入、家长反馈查阅、订单补录、操作日志 |
| ADMIN | 管理员 | 教学科目、价格表、全局配置、学生课时调整、操作日志 |

### 2.2 多角色与会话

- 同一用户可绑定 **多个角色**；登录后在头部下拉 **切换当前生效角色**，侧边菜单与首页仪表盘按 **当前角色** 过滤。
- 会话：`eduflow:session_user`、`eduflow:active_role` 等本地存储；登录成功后设置简要 Cookie（`eduflow_auth`）供扩展中间件使用。

### 2.3 账号状态

- **PENDING**：待审核 —— **禁止登录**，提示等待审核。
- **APPROVED**：已审核 —— 允许登录。
- **REJECTED**：已驳回 —— 禁止登录，可展示驳回原因。
- 历史数据无 `status` 字段时视为兼容通过。

## 3. 功能需求 — 按角色与模块

### 3.1 认证与注册

- **登录**：手机号匹配本地用户列表；校验账号状态。
- **注册**：区分招生老师、伴学教练等注册入口（路由：`register/sales`、`register/tutor`）；新用户审核逻辑与运营侧用户管理衔接。

### 3.2 通用 — 概览（`/`）

- 按角色展示不同工作台文案与卡片（招生：快捷创建试课单、正课单、录入学生；教练：待上课、待反馈、课时等示意数据；学管/运营/管理员：简化欢迎页）。

### 3.3 招生老师（SALES）

- **订单管理**（`/orders`）：列表与详情（`/orders/[id]`）；创建试课（`/trial-lesson/create` 等）、正课选型与创建（`/regular-course/*`）；订单字段含科目、年级、课时、金额、支付凭证、客服/财务审核批注、排单倒计时相关字段等。
- **学员档案**（`/students`）：列表、新建（`/students/create`）、详情与编辑（`/students/[id]`、`/students/[id]/edit`）。
- **个人设置**（`/profile`）：头像、微信二维码、9800 校区信息、分公司归属、正式支付开关、最小起充课时等个人级业务字段。

### 3.4 伴学教练 / 学管（TUTOR / MANAGER）

- **接单中心**（`/orders/market`、`/orders/market/[id]`）：浏览可抢/可接订单并申请接单。
- **我的学员**（`/my-students` 及子路由）：学员维度管理；**课后反馈**创建/编辑/列表（`my-students/feedback/*`）；学习规划相关页面（`study-plan/create` 等）。
- **我的日历**（`/calendar`）：课表视图。
- **我的收入**（`/my-income`）：与个人相关的收入记录展示（类型含试课费、成交奖励、课时费、管理费；含待结算/已结算状态）。
- **接单设置**（`/order-settings`）：接单偏好或日程相关配置。
- **学管专有**：
  - **伴学教练管理**（`/teachers`、`/teachers/[id]`）。
  - **学习规划书管理**（`/study-plan`、`/study-plan/[planId]`）：上传规划文件（PDF/Word）、审核状态（审核中/已完成）、审核人信息记录。

### 3.5 运营人员（OPERATOR）

- **订单管理**（`/manager-orders`、`/manager-orders/[id]`）：全局订单视图与处置（含审核流、凭证查看等与订单类型相关的运营动作）。
- **退款审核**（`/manager-refund`）：退费申请 **双审流程**（一审、二审、撤回、成功、失败等）；与订单课时冻结、最大可退金额计算快照（`RegularRefundBreakdown`）配合展示；操作日志（`RefundOperationLog`）。
- **用户管理**（`/user-management`、`/user-management/[id]`、`edit`）：用户审核、角色与资料维护。
- **分公司管理**（`/branch-companies`）：分公司名称、负责人、客服信息、启用状态等。
- **财务记录**（`/financial-records`）：充值/退款台账，关联订单与招生老师。
- **伴学教练收入**（`/tutor-income`）：按教练聚合周期汇总（试课费笔数与金额、成交奖励、课时费、管理费、总收入）。
- **家长课后反馈**（`/parent-feedback`）：运营查看家长侧结构化评价。
- **订单补录**（`/order-accord`）：线下或其它渠道订单信息的补录入库（`OrderAccordRecord`）。
- **操作日志**（`/operation-logs`）：关键运营动作追溯。

### 3.6 管理员（ADMIN）

- **教学科目配置**（`/subject-settings`）：科目编号、名称、简介、启用。
- **价格配置表**（`/price-settings`）：与课时费档位等相关定价数据维护。
- **全局配置**（`/global-settings`）：系统接单开关、日订单上限、用户日限购、试课/转正红包在线支付开关、教练响应时限与学生上限、试课日限额、学管团队上限、企业微信客服条目与二维码上传（前端本地预览；真实环境应上传服务端）。
- **学生课时调整**（`/student-hours-adjustment`）：管理员对学生课时的人工校正入口。
- **操作日志**（与运营共用入口）。

### 3.7 家长侧（公开链路）

- **家长课后反馈填写页**（`/p/feedback/[orderId]`）：星级评分、多维度结构化问卷（课堂效果、教师表现、规范、环境等）、备注提交。

## 4. 核心业务对象与规则摘要

### 4.1 订单

- **类型**：试课（TRIAL）、正课（REGULAR）。
- **状态（节选）**：草稿、待支付、待客服审核、待财务审核、排单中、待接单、已分配、进行中、已完成、取消相关状态、已退款；退费流程中可标记 **课时冻结**（`refundFreezeActive`），冻结期间原型层提示剩余课时不可用于排课消耗。
- **扩展**：线上线下试课支付方式、支付/G 账号充值/财务审核等多类凭证（Base64 数组）、排单开始时间与超时逻辑字段、是否需要代充鼎伴学费用等。

### 4.2 退费申请

- **种类**：试课、正课、续课、仅转正红包等（`RefundKind`）。
- **状态**：待一审、待二审、一审驳回、撤回、退款成功、退款失败等；记录一审二审人、时间、备注及对招生老师的对外说明字段；可选执行状态（待执行/成功/失败）。

### 4.3 课堂与反馈

- **Lesson**：关联订单、教师、学生、时间段、时长；反馈关联键。
- **LessonFeedback**：教练课后文字反馈、表现评分、作业与下节课计划。
- **LessonFeedbackRecord**：更贴近运营台账的长反馈记录（出勤、作业、扣课时、家长反馈嵌套对象等）。

### 4.4 教练日历档期

- **TeacherCalendarSlot**：可接单 / 已预约 / 锁定（课前后缓冲）。

### 4.5 收入

- **IncomeRecord**：类型（试课费、成交奖励、课时费、管理费）、数量、单价、金额、结算状态、关联订单/反馈/学员等。
- **TutorIncomeSummary**：按教练、按周期聚合统计。

### 4.6 辅助主数据

- **Subject**：教学科目。
- **BranchCompany**：分公司及对接客服。
- **TeacherAccordRecord**：老师数据补录。
- **FinancialRecord**：财务侧充值/退款流水。

## 5. 非功能需求（基于现状）

| 类别 | 要求 |
|------|------|
| 易用性 | 响应式布局；侧栏 + 顶栏；常用操作用仪表盘快捷入口 |
| 性能 | 本地数据规模受限于浏览器；无服务端分页 |
| 安全 | 原型级：无真实密码学认证与服务器鉴权；敏感数据不落盘服务端 |
| 可维护性 | TypeScript 类型集中于 `types/`；存储键集中于 `STORAGE_KEYS` |
| 兼容性 | 依赖现代浏览器 localStorage 与 ES 模块 |

---

## 6. 数据持久化与约束

- 所有业务列表与详情均从 **localStorage** 读写；同机多标签可共享存储。
- 图片类凭证以 **Base64** 形式存储时，需注意配额与性能问题。
- **无后端** 意味着：并发编辑、权限细粒度校验、合规审计仅能作为未来扩展描述，非本仓库交付范围。

---

## 7. 验收与追溯建议

- 以 `components/layout/DashboardLayout.tsx` 中 **导航项 × 角色矩阵** 为菜单可见性验收基线。
- 以 `types/index.ts` 中枚举与接口为 **字段级** 验收基线（订单状态机、退费状态、收入类型等）。
- 主要用户故事路径建议至少覆盖：**注册 → 审核 → 登录 → 招生下单 → 运营审核 → 教练接单 → 日历排课 → 课后反馈 → 收入台账 → 退费双审**。
